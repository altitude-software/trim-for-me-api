---
name: confirm-before-edit
description: Confirmar el enfoque con el usuario antes de hacer ediciones al código
metadata:
  type: feedback
---

Antes de empezar a editar archivos, explicar el plan y esperar confirmación del usuario.

**Why:** El usuario interrumpió ediciones porque quería discutir el enfoque primero (ej. crear un DTO antes de modificar el use case, o confirmar qué archivos se van a tocar).

**How to apply:** Cuando se va a hacer un cambio que involucra múltiples archivos o una decisión de diseño (como dónde poner lógica, qué DTO usar, qué capa modificar), describir el plan primero y esperar que el usuario dé el visto bueno antes de editar.
